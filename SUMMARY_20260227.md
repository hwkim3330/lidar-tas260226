## Final Conclusions
- Operationally stable single-LiDAR gate open is `>=146us` (recommended `150us`).
- `open=28~50us` is not robust in current path; phase-sensitive and unstable over reruns.
- Best long-run setting in this repo:
  - cycle: `781250ns`
  - entries: `305625 / 150000 / 325625 ns` (close/open/close)
  - phase offset: `180000ns`
  - phase_lock: `false`
- Current packet profile is not legacy-3392B path:
  - `udp_profile_lidar=RNG19_RFL8_SIG16_NIR16`
  - UDP payload size is `3328B` (fixed across tested modes with same profile and cpp=16).

## Packet / IFG Understanding (Current Sensor)
- Packet interval depends on mode pps, packet size does not (under same profile/cpp).
- For payload `3328B` and MTU 1500:
  - IP payload fragments: `[1480, 1480, 376]`
  - 1GbE on-wire serialization (incl. IFG, VLAN assumption): `28.176us` total
